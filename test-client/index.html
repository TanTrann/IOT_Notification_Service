3Kết nối MQTT

Sau khi kết nối, client tự subscribe cả 2 topic — mọi bản tin trên broker (kể cả từ thiết bị/server thật nếu đang chạy) sẽ hiện ở Log với tag 📥.
Chưa kết nối

4Giả lập ESP32 → xmini/sensor_data

Payload snake_case đúng format thiết bị xmini; device_id lấy từ bước 2. Service tạo thông báo cho mỗi số đo (không còn ngưỡng); gửi lặp cùng giá trị thì bỏ qua — đổi giá trị (vd bấm "Đất ẩm lại") mới báo tiếp. Nếu server điều khiển .NET đang chạy trên cùng broker, nó cũng nhận bản tin này và có thể tự phát lệnh thật xuống xmini/control → demo được trọn vòng.

5Giả lập server .NET → xmini/control

Đúng format server điều khiển publish: { command, commandId, parameters } — không có device_id. Service gán lệnh cho device của bản tin sensor gần nhất (hoặc DEFAULT_DEVICE_ID trong .env), nên hãy publish 1 bản tin sensor ở bước 4 trước. Mỗi lần bấm sinh commandId mới → thông báo mới; muốn test chống trùng thì bấm "Publish payload này" 2 lần (giữ nguyên commandId) — lần 2 service sẽ bỏ qua.
